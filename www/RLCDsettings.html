WiFi

Transmit power the board uses once connected. Lower values draw less peak current during transmit bursts, which matters if the board resets or drops offline on a marginal power supply (battery/wall charger rather than a laptop's regulated USB port). The board is usually close to the router, so range is rarely the limiting factor.

The network name and password aren't editable here — they're compiled into the board's firmware (secrets.h), not pushed from the Pi. Changing them from this page would risk stranding the board on a network it can no longer reach the Pi from, with no way back except re-flashing over USB.
